# 決算ナビ — 決算短信自動取得・決算日程管理アプリ

日本株に投資する個人投資家向けの、**決算発表予定の確認**と**決算短信PDFの自動取得・閲覧**を効率化するWebアプリのMVPです。

[要件定義](docs/requirements.md) に基づき、MVPスコープ（決算日程一覧・時価総額での絞り込み・マイ銘柄登録・決算短信の自動取得と閲覧）を実装しています。

## 🌐 GitHub Pages で今すぐ試す

**https://kojit1229.github.io/stock_analyze/**

サーバ不要の静的アプリとして全機能（決算予定の絞り込み・マイ銘柄登録・決算短信の確認・PDF閲覧）が動作します。

### 実データの仕組み

[GitHub Actions](.github/workflows/update-data.yml) が平日1日5回、以下の公開データを取得して `frontend/data/*.json` にコミットし、アプリがそれを読み込みます（ヘッダに「実データ」バッジが表示されます）。

| データ | 出典 |
|---|---|
| 銘柄マスタ（コード・銘柄名・市場区分・33業種） | JPX 東証上場銘柄一覧 (`data_j.xls`) |
| 決算発表予定日 | JPX 決算発表予定日ページの公表Excel |
| 決算短信・訂正短信・業績予想修正・配当予想修正・自己株式取得・決算説明資料 | TDnet 適時開示（やのしんWebAPI経由）。PDFはTDnetの実URLを開きます |
| 時価総額 | Yahoo Finance（取得できなかった銘柄は「-」表示となり時価総額絞り込みの対象外） |
| 財務数値の推移（売上高・営業利益・純利益・EPS、年次/四半期） | Yahoo Finance（1回の更新で400銘柄ずつ巡回取得し `financials.json` に蓄積） |
| 貸借対照表・キャッシュフロー（流動/固定資産・流動/固定負債・純資産・投資有価証券、営業/投資/財務CF、年次） | Yahoo Finance（上と同じ巡回で取得。銘柄分析タブのネットキャッシュ比率・BS構成・CF分析で使用） |
| 決算短信の開示履歴アーカイブ（過去2年分・銘柄分析タブで使用） | TDnet（毎時の [backfill ワークフロー](.github/workflows/backfill.yml) が過去方向へ段階構築し `data/history/` のシャードに蓄積。2年分に到達すると自動停止） |
| 決算短信PDFの恒久保存 | [`config/pdf_watchlist.json`](config/pdf_watchlist.json) の銘柄は、PDF本体を `frontend/pdfs/` に自動保存（TDnetの掲載期間後・5年後も閲覧可能。容量制限のため50銘柄までを推奨） |
| 終値・前日比・52週高安・出来高・配当利回り | Yahoo Finance（時価総額と同じバッチで取得し `prices.json` に保存。マイ銘柄タブの終値表示・スクリーナー・アラート判定で使用） |
| マイ銘柄の終値アラート | 平日引け後、[`scripts/generate_alerts.py`](scripts/generate_alerts.py) がマイ銘柄の終値をチェックして `alerts.json` を生成（ホーム画面に表示）。種別: 株価変動±X%・52週高安・出来高急増・決算前日/当日・重要開示・3日連続続落/続伸・決算への反応±X%。メール通知ONの場合はGitHub Issueを作成し@メンション → GitHubの通知メールが届く |
| イベント→株価反応の記録 | 平日引け後、[`scripts/track_reactions.py`](scripts/track_reactions.py) が決算短信・業績/配当予想修正・自己株式取得の開示と株価イベント（急変動±8%・52週高安・出来高急増）について「当日と翌営業日の反応%」を `reactions.json` に蓄積（銘柄分析タブに表示。15時以降の開示は翌営業日を当日として扱う） |

マイ銘柄・閲覧済みフラグ・コメント・アラート設定はブラウザの localStorage に保存されます。**⚙ 設定**画面でGitHubトークン（このリポジトリのContents権限のみのFine-grained PAT）を登録すると、これらを `config/user_data.json` へ**自動コミット（gitバックアップ）**します。アラート機能はこのバックアップを読んで動作するため、アラートを使う場合は自動バックアップの有効化が必要です。手動のJSONファイルバックアップ/復元（マイ銘柄画面）も引き続き使えます。`data/*.json` が無い環境では従来どおりサンプルデータで動作します（「デモ」バッジ）。

このリポジトリの GitHub Pages は「ブランチ(`main`)からのデプロイ」で設定されており、`main` の内容がそのまま配信されます。リポジトリ直下の [`index.html`](index.html) は `frontend/index.html` へリダイレクトするだけのスタブで、実体は [`frontend/`](frontend) です（Pythonサーバモードと共通）。`.nojekyll` を置き、GitHub標準のJekyll処理（READMEの自動トップページ化など）を無効化しています。

> 免責: 表示されるデータはJPX・TDnetの公開情報を機械的に収集したものであり、正確性・完全性は保証されません。投資判断は必ず一次情報をご確認ください。

## 特徴

- **依存ライブラリゼロ** — Python 標準ライブラリ（`http.server` + `sqlite3`）のみで動作。`pip install` 不要。
- **バニラJS SPA** — ビルド不要のフロントエンド。
- **2つの動作モード** — Pythonサーバモード（SQLite永続化）と静的デモモード（GitHub Pages / localStorage）。`/api/*` が見つからない環境では自動で静的モードにフォールバック。
- 4つの主要画面 + 銘柄詳細 + 決算短信詳細（PDFビューア）。

## 動作環境

- Python 3.8+（3.11 で検証）
- モダンブラウザ

## 起動

```bash
python run.py            # http://127.0.0.1:8000 で起動
python run.py --port 9000
```

初回起動時に SQLite DB を初期化し、代表的な日本株24銘柄のサンプルデータ
（時価総額は要件4.2の全レンジを網羅）と決算予定を投入します。

## 使い方

1. **決算予定** 画面で「今日 / 明日 / 今週 / 来週 / 1ヶ月」や時価総額レンジ、
   銘柄コード・銘柄名・市場・業種で絞り込み。列ヘッダで並び替え。
2. 気になる銘柄を **＋登録** して「マイ銘柄」に追加。
3. ヘッダの **⟳ 取得**（またはマイ銘柄画面の取得ボタン）で、登録銘柄のうち
   決算発表日を迎えたものの**決算短信PDFを自動取得**。
4. **決算短信** 画面でPDFを閲覧。開くと閲覧済みになり、コメントを保存できます。
   マイ銘柄の**決算短信履歴のまとめて取得（2年分）**と、**決算短信PDFのGitHub保存**
   （PDF恒久保存リストへの追加＋保存ワークフローの起動。⚙設定のトークンが必要）も
   この画面から実行できます。
5. **銘柄分析** 画面で銘柄を選ぶと、決算数値の推移チャート（売上高・営業利益・
   純利益・営業利益率・EPS、年次/四半期切替）と決算短信の履歴を確認できます。
   財務健全性も分析できます — **ネットキャッシュ比率（清原達郎式:
   (流動資産＋投資有価証券×70%−負債合計)÷時価総額）**、**貸借対照表の構成**
   （流動/固定資産・流動/固定負債・純資産の比率を箱グラフで表示、自己資本比率つき）、
   **キャッシュフロー分析**（営業/投資/財務CFの＋/−と8類型のパターン判定、FCF推移）。
   分析コメントも保存できます。「まとめて取得」で選択したマイ銘柄の
   **決算短信履歴（直近2年分）** を一括取得できます。
6. **銘柄比較** 画面で最大4銘柄の業績・指標（時価総額・売上高・利益・利益率・
   EPS・前期比・次回決算予定・分析コメント）をテーブルとチャートで比較できます。
7. **決算予定**画面の「⭐ マイ銘柄のみ」チップと重要度フィルタ（★n以上）、
   **マイ銘柄**画面の重要度フィルタで、登録銘柄・重要度による絞り込みができます。
   マイ銘柄には当日終値と前日比も表示されます。
8. **⚙ 設定** 画面で、重要度ごとの終値アラート条件（株価変動±X%・52週高値/安値・
   出来高急増・決算前日/当日・重要開示・3日連続±・決算反応±X%）とメール通知、
   gitへの自動バックアップを設定できます。アラートはホーム画面に表示されます。
9. **スクリーナー** 画面で全銘柄を財務指標（時価総額・PER・ネットキャッシュ比率・
   自己資本比率・ROE・売上CAGR・配当利回り）で絞り込み・ランキングできます。
   「清原式割安」「高成長」「高ROE割安」「健全×高配当」のプリセットつき。
11. **市況** 画面で当日の市場全体を確認できます — 値上がり/値下がり銘柄数、
    平均騰落率（単純/時価総額加重）、騰落レシオ（終値履歴から最大25日で算出）、
    セクター別騰落（33業種）、売買代金ランキング、値上がり/値下がり率ランキング、
    52週高値/安値更新（年初来・上場来の代替）、指数インパクト（時価総額増減額）、
    出来高急増、市場区分別騰落、今週の決算発表件数。
10. **銘柄分析** 画面には、決算進捗率（前期通期に対する消化率を前年同時点と比較）、
    ROE・ROA・配当利回り・営業CF÷純利益（利益の質）、イベント（決算発表・業績修正・
    急変動など）への株価反応の記録も表示されます。

> 注: MVPでは外部データソース（TDnet等）へは接続せず、`MockDisclosureSource`
> がサンプルPDFを生成します。取得ロジックは `DisclosureSource` インターフェースで
> 差し替え可能な設計のため、将来 TDnet 実装へ置き換えられます。

```
  market_cap.py         時価総額レンジ定義・判定（要件4.2）

  seed.py               サンプル銘柄・決算予定
  fetcher.py            決算短信の自動取得（差し替え可能なソース）
  pdfgen.py             最小PDFジェネレータ（サンプルPDF生成）
  api.py                APIハンドラ
  server.py             HTTPサーバ・ルーティング・静的配信
frontend/
  index.html / app.js / styles.css   バニラJS SPA
  local-api.js          静的モード用のブラウザ内API (GitHub Pages 用)
tests/test_api.py       統合テスト（サーバを起動して検証）
scripts/smoke.py        起動スモークテスト
scripts/check_local_api.mjs  静的モードの回帰チェック (Node)
```
